** 코딩 순서 **
1)내용 작성
2)마크업 : 용도나 목적에 맞는 태그 작성
3)콘텐츠 블럭
4)css 초기화(reset) : 태그들이 가지고 있는 기본 속성과 값을 코딩하기 편한 값으로 변경
5)css 작성
6)스크립트 적용
7)프로그래밍 적용
8)표준 접근성

1. header
- 페이지의 독자적 요소
- 로고, 네비게이션
- 반드시 하나 이상의 제목(h1~h6) 명시
- section, header, footer 태그 사용 불가
- 요소의 위치 무관

2. article
- 페이지의 내용 상 별개의 내용을 구성할 때 사용되는 섹션

3. section
- 단락, 이미지 제목 등의 콘텐트를 묶어 더 큰 논리적 단위
- body 안에 들어갈 수 있는 모든 요소 사용 가능(header, footer포함)
- class나 id 속성을 사용하여 섹션에 대한 특정정보(메타데이터) 추가

4. main 
- 문서의 주요 내용 정의(요소 내부의 내용은 문서에 고유)
- 문서 내에 한번만 사용
- article, aside, header, nav, footer 요소의 하위에 사용 불가
- 사이드 콘텐트, 저작권 정보, 사이트 로고, 검색 양식 등 모든 내용 포함 가능

5. nav
- 페이지 링크, 내부 링크가 있는 섹션 마크업 시 사용
- 메인 네비게이션이 있는 섹션에만 사용

6. aside
- 본문 내용에 대한 추가적인 설명, 부수적 콘텐츠 표시에 사용하는 태그

7. footer
- 저작권, 개인정보 보호정책, 연락처 등의 콘텐츠
- 문서상의 위치 무관

8. div
- div는 아무런 의미가 없어 어떠한 내용도 담을 수 있다.
